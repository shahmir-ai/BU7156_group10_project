## What This Notebook Does

This notebook documents three structured LLM prompt experiments. Each experiment follows the same workflow: design a prompt → run it → critique the output against verified data → refine the prompt → run again → compare V1 vs V2. The goal is to demonstrate critical and iterative use of LLMs, not blind reliance.

---

## What's Inside

### Prompt 1 — Insight Synthesis
| Section | What It Covers |
|---------|---------------|
| Purpose | Test whether an LLM can convert EDA statistics and model results into a CEO-readable briefing without introducing errors |
| V1 | Unconstrained prompt; LLM fabricated three euro sub-figures (€2.3M + €0.9M + €3.9M = €7.1M) and omitted the 153-customer high-risk segment |
| V2 | Added explicit constraints; all three errors eliminated. New issue: LLM speculated about salary interaction effects without evidence |
| Key finding | LLMs produce plausible-sounding but fabricated financial figures. Prompt constraints fix obvious errors but introduce subtler unverifiable claims |

### Prompt 2 — Code Generation (SHAP Analysis)
| Section | What It Covers |
|---------|---------------|
| Purpose | Test whether an LLM can generate correct SHAP analysis code for a trained Random Forest model |
| V1 | Generated working beeswarm code, but the plot format hides discrete product threshold structure and cannot show feature interactions |
| V2 | Added dependence plot + mean SHAP table + Germany × gender subgroup analysis. Revealed MDI overstates Salary (2nd → 7th in SHAP) and Credit Score (3rd → 8th) |
| Key finding | German female churn (37.6% in EDA) appears driven by country, not gender — SHAP gender gap within Germany is only 0.0022 |

### Prompt 3 — Critical Review
| Section | What It Covers |
|---------|---------------|
| Purpose | Test whether an LLM can function as a useful adversarial reviewer of our team's six analytical conclusions |
| V1 | Unconstrained review; produced 21 counter-arguments (13 valid, 4 unverifiable, 2 incorrect). Made two factual errors: assumed salary–balance collinearity (actual r = 0.013) and missed that we already compared three models |
| V2 | Constrained to 12 available variables only; identified gender as the biggest omission in our conclusions (Female 25.1% vs Male 16.5%) |
| Key finding | 46% of the high-value segment (age 40–60, balance >€100k) is German — conclusions 2 and 3 partially describe the same effect |

## LLM Model Used

All prompts were run using **Claude Sonnet (claude.ai)**.

## Key Findings for the Team

| Finding | Source | Impact |
|---------|--------|--------|
| MDI feature importance overstates Salary (2nd in MDI → 7th in SHAP) and Credit Score (3rd → 8th) | Prompt 2 | Modellers: consider presenting SHAP alongside MDI in slides |
| German female churn (37.6%) driven more by country than gender (within-Germany SHAP gap = 0.0022) | Prompt 2 | Storytellers: qualify gender claims in the narrative |
| High-value segment confounded with Germany (46% of segment is German) | Prompt 3 | Storytellers: acknowledge this overlap in recommendations |
| Gender (F=25.1% vs M=16.5%, 8.6pp gap) absent from all six conclusions | Prompt 3 | Storytellers: consider adding gender to final narrative |
| LLM fabricated sub-figures summing to the correct total | Prompt 1 | All: never trust LLM-generated numbers without verification |

## How to Read the Notebook

Each prompt section follows this structure:

1. **Purpose** — what we're testing and why
2. **V1 Prompt** — exact text sent to the LLM
3. **V1 Output** — full LLM response
4. **V1 Accuracy Critique** — line-by-line verification table (✅/❌/⚠️)
5. **Iteration Rationale** — what went wrong in V1 and what we changed
6. **V2 Prompt** — refined prompt with added constraints
7. **V2 Output** — full LLM response
8. **V2 Accuracy Critique** — same verification table format
9. **Comparison** — V1 vs V2 side-by-side, with lessons learned
10. **GenAI Acknowledgement** — model used, who verified, what was excluded
